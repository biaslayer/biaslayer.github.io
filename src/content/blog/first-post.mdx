## Daily writing needs low friction

Most personal blogs fail at the moment between thinking and publishing.

The idea is fine. The draft is half-there. But the actual workflow asks for too many little decisions: page templates, routing, metadata, previews, broken styles, and build steps that feel bigger than the post itself.

That is why a static MDX pipeline matters. It turns publishing into file creation instead of project management.

## The archive should still feel intentional

Speed does not have to make the site feel disposable. A strong blog shell does a lot of quiet work:

- It keeps the navigation predictable.
- It makes older posts easy to rediscover.
- It gives each article enough whitespace to breathe.
- It keeps typography consistent so the writing carries the weight.

The result is a workflow where the author moves quickly, but the reader still gets a premium experience.

## Good defaults compound

If the site calculates reading time, builds the table of contents, updates the sidebar, and ships syntax highlighting automatically, then every new post starts from a better baseline.

That is the real value of this kind of setup. It is not just static. It is composable routine.

```ts
export function publish(post: Draft) {
  return {
    ...post,
    status: "ready",
    friction: "low"
  };
}
```
